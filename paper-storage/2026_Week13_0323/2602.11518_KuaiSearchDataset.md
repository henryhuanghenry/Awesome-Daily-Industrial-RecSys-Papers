## Abstract

E-commerce search faces challenges including highly ambiguous queries, noisy product texts with weak semantic order, and diverse user preferences. Existing e-commerce search datasets suffer from notable limitations: queries are often heuristically constructed, cold-start users and long-tail products are filtered out, query and product texts are anonymized, and most datasets cover only a single stage of the search pipeline.

To address these challenges, we construct and release **KuaiSearch** — to the best of our knowledge, the **largest e-commerce search dataset currently available**. KuaiSearch is built upon real user search interactions from the Kuaishou platform:
- **330,000 users**
- **18,000,000 products**
- **2,500,000 real search queries**

Systematically spanning three key stages: **recall**, **ranking**, and **relevance judgment**. Preserves authentic user queries and natural-language product texts, covering cold-start users and long-tail products.

## 1. Introduction

E-commerce search is the primary interface for users to express information needs. Key challenges:
1. **Highly ambiguous queries**: brand names, product categories, attribute keywords (not full descriptions)
2. **Noisy product texts**: merchants inject irrelevant keywords for exposure; core attributes appear in unstructured order
3. **User heterogeneity**: same query → different budget ranges, preferences, usage contexts

**Limitations of existing datasets**:
- **Amazon datasets**: queries heuristically constructed; cold-start/long-tail filtered out
- **JDsearch**: real queries but texts are anonymized → constrains LLM research
- **Qilin**: closer to content browsing than product search; only 1 day of data; lacks purchase intent signals

**KuaiSearch contributions**:
1. Real user queries, full-category product catalog, diverse long-term user behaviors, no popularity filtering
2. Covers three core components: recall, ranking, and relevance judgment
3. Comprehensive dataset analysis and benchmark evaluation

## 3. The KuaiSearch Dataset

Data construction pipeline:

1. **User Behavior Collection**: 331,930 users with core interaction behaviors after June 1, 2025; complete search activity including clicks and purchases; negative feedback (no clicks) preserved

2. **Product Metadata Collection**: product titles, three-level category hierarchies, brand names, seller names (all in **plaintext**)

3. **User Metadata Collection**: gender, age group, geographic location

4. **Ranking Data Construction**: 
   - User side: followees/followers count, past 30-day clicks/exposures/purchases/GMV
   - Product side: exposure count, click count, purchase count, price
   - Recent 20 clicked products + recent 20 purchased products as sequential behavior features

5. **Relevance Data Construction**: 46,422 query–product pairs, annotated by domain experts with **four-level graded relevance**

**KuaiSearch Statistics:**

| Component | Scale |
|-----------|-------|
| Users | 331,930 |
| Products | ~18,000,000 |
| Queries | ~2,500,000 |
| Relevance Pairs | 46,422 |

Also released: **KuaiSearch-Lite** (subset for rapid validation under constrained budgets).

## 4. Primary Data Analysis

### 4.1. Demographics

- 331,930 users from 62+ countries (99.82% in China)
- Gender: 59.34% female
- Age distribution: 31–40 (24.96%), 41–49 (16.18%), 18–23 (15.94%), 50+ (15.90%)
- Core user groups (12–50): >80% of population

### 4.2. Search Interaction and Content Analysis

**Product Analysis:**
- Power-law distribution: most products have few interactions (cold products dominate)
- Clear long-tail distribution

**User History Analysis:**
- Substantial heterogeneity: some users rarely shop, others have extremely rich histories

**Query Analysis:**

*Table 3: Engagement statistics across different query lengths:*
- Medium-length queries (5–8 chars): 44.68% of all requests
- SessCTR: 0.496–0.531 (no clear monotonic trend)
- Very long queries (≥10): highest SessCTR
- AvgClick increases steadily with query length (1.06 → 1.33)
- ItemCTR peaks at short-to-medium lengths (3–4 chars) then declines

*Table 4: Engagement statistics across search entrances:*
- Homepage + Mall: >93% of requests, higher SessCTR (explicit intent)
- Placeholder: low SessCTR (intent exploration stage)
- Detail page: higher AvgClick and ItemCTR (comparison/choice expansion)
- Related-feed: highest ItemCTR and AvgClick (focused explicit intent)

*Query category distribution (top 12):*
Apparel dominates (women's clothing, men's clothing, footwear), followed by mobile devices, home necessities, personal care, beauty products. ~78 first-level categories total.

### 4.3. Relevance Data Analysis

**Four-level graded relevance scheme:**
- **0**: Clearly irrelevant (category/attribute mismatch)
- **1**: Weakly relevant (related category but fails to satisfy intent)
- **2**: Partially relevant (correct category but violates key attribute/brand constraints)
- **3**: Highly relevant (fully satisfies user's search intent)

Distribution: Labels 2 and 3 account for majority → challenging, fine-grained benchmark.

## 5. Experiment

All experiments conducted on **KuaiSearch-Lite** for efficiency.

### 5.1. Recall Task Performance

- **Test set**: Last day of interaction logs
- **Metrics**: Recall@K (R@K) and Hit Rate@K (HR@K)
- **Baselines**: BM25, DocT5Query, DPR-ADE, DPR-SDE, DSI, LTRGR

**Key findings from Table 6:**
1. **Embedding-based retrieval >> lexical matching and generative retrieval**
2. DPR-SDE > DPR-ADE (shared encoder mitigates embedding space misalignment)
3. Generative retrieval (DSI, LTRGR) performs similarly to BM25 — complex e-commerce queries make precise identifier generation challenging
4. LTRGR marginally improves HR metrics via ranking loss (still significantly weaker than DPR)

### 5.2. Ranking Task Performance

- **Test set**: Last day of ranking data
- **Metrics**: Logloss and ROC-AUC
- **Baselines**: DNN, Wide & Deep, DCN, DCN-v2, DIN

**Key findings from Table 7:**
1. DNN achieves lowest Logloss (superior calibration, simple structure avoids overfitting)
2. DIN best ROC-AUC (attention captures dynamic user interests)
3. DCN-v2 consistently outperforms DCN (AUC: 0.6194 → 0.6239)
4. Small performance gaps across architectures → future improvements likely from richer feature engineering

### 5.3. Relevance Task Performance

- **Split**: 90% train / 10% test; relevance ≥3 → positive
- **Metrics**: ROC-AUC and PR-AUC
- **Baselines**: BGE (bi-encoder), BERT-Chinese/Multilingual, XLM-RoBERTa (cross-encoder), Qwen3, Llama3.2 (LLM-based)

**Key findings from Table 8:**
1. **LLM-based generative classification > cross-encoder > bi-encoder**
2. Qwen3-1.7B achieves highest ROC-AUC and PR-AUC
3. Qwen3-0.6B surpasses Llama3.2-1B (strong parameter efficiency of Qwen3 family)
4. Models capturing fine-grained semantic interactions between queries and items perform better

---

## 6. Conclusion

KuaiSearch is the largest e-commerce search dataset available:
- Authentic logs from Kuaishou platform
- Real user queries + natural-language product texts
- No popularity filtering → long-tail and cold-start users preserved
- Three pipeline stages: recall, ranking, relevance judgment
