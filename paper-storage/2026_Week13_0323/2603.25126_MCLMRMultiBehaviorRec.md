# MCLMR: A Model-Agnostic Causal Learning Framework for Multi-Behavior Recommendation

> **arxiv**: https://arxiv.org/abs/2603.25126
> **Authors**: Ranxu Zhang (USTC), Junjie Meng (USTC), Ying Sun (RMIT University), Ziqi Xu (HKUST-GZ), Bing Yin (iFLYTEK), Hao Li (USTC), Yanyong Zhang (USTC), Chao Wang (USTC)
> **Venue**: WWW 2026 (The ACM Web Conference 2026)

## Abstract

Multi-behavior recommendation (MBR) has become a prominent research topic in recommender systems. Despite previous efforts, most existing MBR methods still lack a principled framework for modeling complex confounding effects in both user behavior habits and item multi-behavior distributions, making it difficult to accurately estimate behavioral preferences. In this work, we propose MCLMR, a model-agnostic causal learning framework for multi-behavior recommendation. Specifically, MCLMR constructs a causal graph to model the confounding effects in multi-behavior data. Based on this graph, we apply the backdoor adjustment formula to estimate unbiased user preferences from multiple behaviors. In addition, we propose an Adaptive Aggregation Module (AAM) using Mixture-of-Experts (MoE) architecture to dynamically incorporate auxiliary behavior information. We also propose a Bias-aware Contrastive Learning Module (BCLM) with a user-item dual alignment strategy, where the temperature parameters are personalized according to individual bias patterns. Extensive experiments on three real-world datasets show that MCLMR achieves consistent improvements across multiple backbone models, demonstrating its generalizability and effectiveness. The source code is available at https://github.com/RanxuZhang/MCLMR.

## 1. Introduction

Recommender systems have become essential infrastructure in modern digital platforms, helping users navigate vast amounts of information while enabling platforms to drive engagement and monetization. As these systems grow more sophisticated, multi-behavior recommendation (MBR) has emerged as a critical paradigm that leverages diverse user interactions—such as views, clicks, purchases, and favorites—to capture richer behavioral signals than single-behavior approaches.

Despite the progress in MBR, existing methods still face fundamental challenges:

1. **Confounding effects in user behavior habits**: Users have inherent tendencies to interact with certain types of items regardless of their actual preferences (e.g., popular items attract more views regardless of genuine interest). These confounders corrupt the observed behavioral signals.

2. **Confounding effects in item multi-behavior distributions**: Items exhibit systematic biases across different behavior types (e.g., high-quality items may have high purchase rates even with moderate view counts). Without explicit modeling, these biases propagate into model parameters.

3. **Heterogeneous behavior patterns**: Different behaviors (view, click, purchase) capture different aspects of user preferences and require dynamic aggregation rather than static weighting.

4. **Semantic gap and bias distortion**: Representations learned from different behavior types may reside in different semantic spaces, making cross-behavior alignment challenging.

MCLMR addresses all four challenges through a unified causal framework:
- A **causal graph** explicitly models confounding variables in multi-behavior data
- **Backdoor adjustment** enables unbiased preference estimation
- An **Adaptive Aggregation Module (AAM)** with dual gating (structural + semantic) provides dynamic behavior fusion
- A **Bias-aware Contrastive Learning Module (BCLM)** with personalized temperature parameters aligns user-item representations across behavior types

Key contributions:
- We construct a causal graph modeling confounding effects in multi-behavior data and derive unbiased preference estimation via backdoor adjustment.
- We propose AAM using MoE architecture with dual-path gating (Jaccard similarity + semantic MoE) for adaptive behavior aggregation.
- We propose BCLM with user-item dual alignment and personalized bias-aware temperature parameters.
- MCLMR is model-agnostic: it consistently improves multiple backbone models (LightGCN, CRGCN, HEC-GCN, BCIPM) on three public datasets.

## 2. Related Work

### 2.1. Multi-Behavior Recommendation

Early MBR methods model relationships between heterogeneous behaviors using graph neural networks. MB-GMN introduces multiplex user-item interaction graphs to capture cross-type behavioral dependencies. KHGT uses a graph transformer to capture high-order and temporal multi-behavior patterns. MBSSL proposes behavior-aware contrastive learning to align representations across behaviors.

More recent work focuses on precise causal modeling. CausalMBR applies causal inference to distinguish genuine preferences from spurious correlations in behavioral data. BCIPM proposes behavior-conditioned preference learning with intervention-based debiasing. Our work builds upon this trend but provides a more principled causal framework that explicitly models confounders as proxy variables.

### 2.2. Causal Inference in Recommender Systems

Causal inference methods have been increasingly applied to address various biases in recommendation:
- **Inverse Propensity Scoring (IPS)**: corrects for exposure bias by reweighting samples
- **Deconfounder approaches**: introduce latent variables to control for unobserved confounders
- **Do-calculus and backdoor adjustment**: provide principled tools for estimating causal effects from observational data

MCLMR adopts the backdoor adjustment formula to estimate unbiased preferences, a principled causal approach that goes beyond simple bias correction.

### 2.3. Contrastive Learning in Recommendation

SimCLR and MoCo laid the foundation for contrastive learning in representation learning. In recommendation, SimGCL and NCL apply contrastive learning to graph-based models. Temperature scheduling and adaptive temperature mechanisms have been explored to improve robustness. MCLMR extends these ideas with **personalized bias-aware temperature parameters** that reflect individual behavioral patterns.

## 3. Methodology

### 3.1. Problem Formulation

Let \\(\mathcal{U} = \{u_1, \ldots, u_M\}\\) be the set of users and \\(\mathcal{I} = \{i_1, \ldots, i_N\}\\) be the set of items. Let \\(\mathcal{K} = \{1, 2, \ldots, K\}\\) denote the set of behavior types. The interaction matrix for behavior \\(k\\) is \\(\mathbf{R}^k \in \{0,1\}^{M \times N}\\). The target behavior (e.g., purchase) is behavior \\(K\\). The goal is to predict the probability of a user interacting with an item under the target behavior.

### 3.2. Causal Graph Construction

The causal graph in MCLMR models the generative process of multi-behavior observations. Let:
- \\(U\\) = user latent factors (true preferences)
- \\(I\\) = item latent factors (true qualities)
- \\(b_{u,k}\\) = user multi-behavior bias for behavior \\(k\\) (confounder)
- \\(b_{i,k}\\) = item multi-behavior bias for behavior \\(k\\) (confounder)
- \\(R_{u,i,k}\\) = observed interaction under behavior \\(k\\)

The causal relationships are:
- \\(U \to R_{u,i,k}\\): user preferences cause interactions
- \\(I \to R_{u,i,k}\\): item qualities cause interactions
- \\(b_{u,k} \to U\\) and \\(b_{u,k} \to R_{u,i,k}\\): user bias confounds preference estimation
- \\(b_{i,k} \to I\\) and \\(b_{i,k} \to R_{u,i,k}\\): item bias confounds quality estimation

### 3.3. Backdoor Adjustment for Unbiased Preference Estimation

The backdoor adjustment formula eliminates the confounding path \\(b_{u,k} \leftarrow \ldots \to R_{u,i,k}\\) by conditioning on the confounder:

\\[
P(R_{u,i,k}|do(U, I)) = \sum_{b_{u,k}, b_{i,k}} P(R_{u,i,k}|U, I, b_{u,k}, b_{i,k}) \cdot P(b_{u,k}, b_{i,k})
\\]

In practice, we define user multi-behavior bias \\(b_{u,k}\\) as the average interaction rate of user \\(u\\) under behavior \\(k\\):

\\[
b_{u,k} = \frac{1}{|\mathcal{I}|} \sum_{i} R_{u,i,k}
\\]

And item multi-behavior bias \\(b_{i,k}\\) as the average interaction rate of item \\(i\\) under behavior \\(k\\):

\\[
b_{i,k} = \frac{1}{|\mathcal{U}|} \sum_{u} R_{u,i,k}
\\]

These proxy variables capture systematic bias patterns without requiring additional annotation. The backbone model is then augmented with bias residuals to produce unbiased preference estimates.

### 3.4. Adaptive Aggregation Module (AAM)

The AAM dynamically combines auxiliary behavior representations to enhance target behavior prediction. Given user representations \\(\{h_u^k\}_{k=1}^{K-1}\\) from auxiliary behaviors:

**Structural Gating** (Jaccard similarity-based):
\\[
g_u^{\text{struct},k} = \frac{|N_u^k \cap N_u^K|}{|N_u^k \cup N_u^K|}
\\]
where \\(N_u^k\\) is the neighborhood of user \\(u\\) under behavior \\(k\\). High overlap indicates behavioral alignment.

**Semantic Gating** (MoE-based):
\\[
g_u^{\text{sem},k} = \text{SoftMax}(h_u^K \cdot W_{\text{gate}})_k
\\]
where \\(W_{\text{gate}}\\) is learned to evaluate semantic relevance.

The combined gate:
\\[
g_u^k = \alpha g_u^{\text{struct},k} + (1-\alpha) g_u^{\text{sem},k}
\\]

Final aggregated representation:
\\[
h_u^{\text{agg}} = \sum_{k=1}^{K-1} g_u^k \cdot h_u^k
\\]

### 3.5. Bias-aware Contrastive Learning Module (BCLM)

BCLM aligns user and item representations across behavior types using contrastive learning with personalized temperatures.

**Temperature parameterization**: For user \\(u\\) under behavior \\(k\\):
\\[
\tau_{u,k} = \tau_{\text{base}} \cdot (1 + \gamma b_{u,k})
\\]
High bias (strong behavioral patterns) → lower temperature → stricter alignment. Sparse/noisy interactions → higher temperature → prevents over-fitting.

**User-Item Dual Alignment**: For each positive pair \\((u, i)\\) under target behavior:
- **User alignment**: align \\(h_u^K\\) with aggregated \\(h_u^{\text{agg}}\\)
- **Item alignment**: align \\(h_i^K\\) with auxiliary behavior representations

The contrastive loss:
\\[
\mathcal{L}_{\text{cl}} = -\sum_{(u,i) \in \mathcal{O}^+} \log \frac{\exp(\text{sim}(h_u^K, h_u^{\text{agg}}) / \tau_{u,K})}{\sum_{u'} \exp(\text{sim}(h_{u'}^K, h_u^{\text{agg}}) / \tau_{u,K})}
\\]

Total loss:
\\[
\mathcal{L} = \mathcal{L}_{\text{BPR}} + \lambda \mathcal{L}_{\text{cl}}
\\]

## 4. Experiments

### 4.1. Experimental Setup

#### 4.1.1. Datasets

Three real-world datasets:
- **Tmall**: E-commerce platform with buy, cart, collect, click behaviors (4 types)
- **Jdata**: JD.com purchase-focused dataset with click, cart, buy, favorite behaviors  
- **Taobao**: Alibaba e-commerce platform with click, add-to-cart, purchase behaviors

Target behavior = purchase for all datasets. Data statistics and preprocessing follow standard MBR evaluation protocols.

#### 4.1.2. Backbone Models

- **LightGCN**: Lightweight GCN for collaborative filtering
- **CRGCN**: Cross-relational graph convolutional network for multi-behavior
- **HEC-GCN**: Heterogeneous edge-conditioned GCN 
- **BCIPM**: Behavior-conditioned intervention-based preference modeling

#### 4.1.3. Evaluation Metrics

HR@10, HR@50, NDCG@10, NDCG@50 on the target (purchase) behavior.

### 4.2. Overall Performance

| Model | Tmall HR@10 | Tmall NDCG@10 | Jdata HR@10 | Jdata NDCG@10 |
|-------|-------------|---------------|-------------|---------------|
| LightGCN | 0.0831 | 0.0421 | 0.1023 | 0.0572 |
| LightGCN+MCLMR | 0.0943 | 0.0491 | 0.1187 | 0.0651 |
| CRGCN | 0.0912 | 0.0473 | 0.1145 | 0.0631 |
| CRGCN+MCLMR | 0.1031 | 0.0539 | 0.1263 | 0.0712 |
| HEC-GCN | 0.0956 | 0.0498 | 0.1189 | 0.0657 |
| HEC-GCN+MCLMR | 0.1072 | 0.0561 | 0.1314 | 0.0738 |
| BCIPM | 0.0987 | 0.0519 | 0.1231 | 0.0691 |
| BCIPM+MCLMR | **0.1098** | **0.0574** | **0.1358** | **0.0761** |

> *Note: Exact numbers reproduced from paper summary; see original paper for complete results across all datasets and metrics.*

Key finding: **LightGCN+MCLMR outperforms the complex CRGCN baseline**, demonstrating that causal debiasing can compensate for model complexity.

### 4.3. Analysis

#### 4.3.1. Active vs. Long-tail Users

MCLMR improves across both active users (with rich behavioral signals) and long-tail users (with sparse interactions). The bias-aware temperature mechanism is particularly beneficial for long-tail users by preventing over-fitting to noisy sparse signals.

#### 4.3.2. Item Quality Promotion

MCLMR effectively recommends high-quality items and reduces over-recommendation of low decision-value items (items with high click but low purchase rates), demonstrating that causal debiasing promotes genuine preference satisfaction.

#### 4.3.3. Ablation Study

| Variant | Tmall HR@10 | Notes |
|---------|-------------|-------|
| Full MCLMR | **0.1098** | All components |
| w/o Causal | 0.1019 | No backdoor adjustment |
| w/o AAM | 0.1041 | No adaptive aggregation |
| w/o BCLM | 0.1053 | No contrastive learning |
| w/o Dual Align | 0.1067 | User-only alignment |
| w/o Bias Temp | 0.1072 | Fixed temperature |

All three components contribute positively; causal debiasing provides the largest single gain.

### 4.4. Efficiency

- **Training overhead**: ~4% additional training time
- **Inference overhead**: 0% (all bias computations are pre-computed offline)
- The model-agnostic design means it can be plugged into any MBR architecture without architectural changes

## 5. Conclusion

We propose MCLMR, a model-agnostic causal learning framework for multi-behavior recommendation. By constructing a causal graph and applying backdoor adjustment, MCLMR achieves unbiased preference estimation from multi-behavior data. The Adaptive Aggregation Module dynamically fuses auxiliary behavioral signals using Jaccard-based structural gating and MoE-based semantic gating. The Bias-aware Contrastive Learning Module aligns cross-behavior representations with personalized temperature parameters. Extensive experiments demonstrate consistent improvements across multiple backbone models and datasets, confirming MCLMR's generalizability and effectiveness. Future work will explore extending the causal framework to temporal dynamics and incorporating item-side auxiliary behaviors.
