# Scaling Multilingual Semantic Search in Uber Eats Delivery

> **arxiv**: https://arxiv.org/abs/2603.06586  
> **Authors**: Zheng Liu, Haoyang Chen, Divya Nagar, Luting Yang, Mehul Parsana (Uber Technologies Inc.)  
> **Venue**: Planned for SIGIR / SIGIR eCom / KDD 2026

## Abstract

We present a production-oriented semantic retrieval system for Uber Eats that **unifies retrieval across stores, dishes, and grocery/retail items**. Our approach fine-tunes a **gte-Qwen2 two-tower base model** using hundreds of millions of query-document interactions. We train the model with InfoNCE on in-batch negatives and triplet-NCE loss on hard negatives, leveraging **Matryoshka Representation Learning (MRL)** to serve multiple embedding sizes from a single model. Our system achieves substantial recall gains over a strong baseline across six markets and three verticals. The paper covers end-to-end: data curation, model architecture, large-scale training, and evaluation.

## 1. Introduction

Search is a primary entry point for Uber Eats users. Historically, the system relied on fragmented solutions — either separate BERT-based models for each vertical or simple keyword-based lexical search. We address these limitations by introducing a **single, unified, multilingual, and multi-vertical system** built on a two-tower architecture with gte-Qwen2 backbone.

Our training strategy combines:
1. **Domain adaptation phase**: hundreds of millions of large-scale click and ATC (add-to-cart) logs
2. **Fine-tuning phase**: high-quality relevance data mined by an LLM

**Key contributions:**
- **Unified Two-Tower System**: Single multilingual model for restaurants, grocery, and retail
- **Scalable Training Recipe**: InfoNCE with in-batch negatives + optional Triplet NCE for hard negatives
- **Flexible Embeddings with MRL**: Serve multiple embedding sizes from a single model
- **Comprehensive Evaluation**: Offline evaluations across 6 markets and 3 verticals
- **Productionization and Innovation**: ANN indexing, online serving, model-based distance functions

## 2. Related Work

### 2.1. Two-Tower Architectures

The two-tower (dual-encoder) model is foundational for large-scale retrieval. Separate encoders for query and document allow pre-computation of document embeddings for ANN search. Our system adopts this architecture with unshared weights.

### 2.2. LLMs as Embedding Models

Recent research shows LLMs can be fine-tuned to create highly effective embedding models. Gemini Embedding, NV-Embed, and GTE demonstrate this with state-of-the-art performance. We select gte-Qwen2 as our foundation.

### 2.3. Training Techniques for Dense Retrieval

InfoNCE loss with in-batch negative sampling is effective and scalable. Hard negative mining is critical for refinement. We extend this using an LLM to create high-quality labeled datasets.

### 2.4. Flexible Embedding Architectures

MRL trains a single high-dimensional embedding where initial prefixes are as informative as independently trained lower-dimensional models.

### 2.5. Multi-Modal and Multi-Vertical Embeddings

Training a single model for diverse document types (stores, dishes, grocery items) is a key challenge. Our structured JSON input format allows the model to implicitly understand document type.

### 2.6. Learned Non-linear Scoring

Beyond fixed cosine/dot-product scoring, recent work explores learned similarity heads. We explore a model-based neural distance function for micro re-ranking.

## 3. System Overview

### 3.1. Training and Model Generation

Two-tower DNN on gte-Qwen2 backbone. PyTorch + HuggingFace Transformers + Ray for distributed training. DeepSpeed ZeRO-3 for sharding. Mixed-precision training + gradient accumulation. Hundreds of millions of data points. Versioned artifacts tracked for reproducibility.

### 3.2. Online Query Serving

Optimized for low latency. Query normalized to structured JSON → query tower embedding → MRL dimension selection (128–1536) based on surface and use case. Short-term cache for popular queries.

### 3.3. Offline Inference and Indexing

Documents embedded at feature level, joined back to full catalog of billions of candidates. Automated bi-weekly full retrain + index rebuild. Daily delta pipelines for freshness. Embeddings stored in feature store tables. HNSW-backed indices with FP32 and INT8 representations.

## 4. Methods: Data, Model, Losses, and Training

### 4.1. Data and Features

Training data from large-scale implicit user feedback in Uber Eats search logs (Dec 2023 – Mar 2025). Several hundreds of millions of positive (query, document) pairs.

**Data sources:**
- **Global Search (GS)**: (query, store) clicks
- **In-Store Search—OFD**: (query, dish) clicks
- **In-Store Search—GR**: (query, grocery item) add-to-cart events

Balanced across EN/non-EN languages and verticals.

**Features:**
- **Queries**: JSON blob with search query, country, language, context. Max 128 tokens.
- **Documents**: Vertical-specific JSON schema (names, categories, descriptions, cuisine types, tags). Max 1024 tokens.

### 4.2. Model Architecture

> **Figure 1.** *(inline SVG)* Two-tower gte-Qwen2 architecture: query and document towers produce embeddings scored by cosine or an FFN head; training uses InfoNCE, Triplet NCE, and MRL.

- **Backbone**: gte-Qwen2-1.5B with unshared weights between query and document towers
- **Input**: JSON-formatted strings (significantly outperforms plain text concatenation)
- **Pooling**: EOS token pooling (leverages bidirectional attention)
- **Similarity**: Cosine similarity (default) + optional learned FFN distance function

### 4.3. Objectives and Optimization

**Stage 1 — InfoNCE Mode:**

$$\mathcal{L}_{\mathrm{InfoNCE}} = -\frac{1}{N} \sum_{i=1}^N \log \frac{\exp(s_{ii}/\tau)}{\sum_{j=1}^N \exp(s_{ij}/\tau)} \quad (1)$$

where $s_{ij} = \mathrm{sim}(q_i, d_j)$ (cosine), temperature $\tau = 0.07$, micro batch size 512.

**Stage 2 — Triplet NCE Mode** (hard positive/negative examples mined from logs and verified by LLM):

$$\mathcal{L}_{\mathrm{Triplet\text{-}NCE}} = \mathbb{E}_{(q,d^+) \sim \mathcal{P}} \left[\log\left(1 + \exp(-z_{qd^+})\right)\right] + \mathbb{E}_{(q,d^-) \sim \mathcal{N}} \left[\log\left(1 + \exp(+z_{qd^-})\right)\right] \quad (2)$$

**Matryoshka Representation Learning (MRL):**

$$\mathcal{L}_{\mathrm{MRL}} = \sum_{m \in \mathcal{M}} c^{(m)} \mathcal{L}_\star\left(E^{(m)}(q), E^{(m)}(d)\right) \quad (3)$$

with $\mathcal{M} = \{128, 256, 512, 768, 1024, 1280, 1536\}$, $c^{(m)} = 1$.

**Optimizer**: Adam, fixed LR $5 \times 10^{-5}$, no warmup. Only fine-tune last 3 layers to prevent catastrophic forgetting.

### 4.4. Training Setup at Scale

Uber's internal Michelangelo Canvas platform. DDP (NCCL) + Ray + DeepSpeed ZeRO-3. Mixed precision (bf16 on H100). Typical run: 24–48 hours on 8×H100. Experiments tracked in Comet.

## 5. Evaluation

### 5.1. Models and Variants Compared

Baselines: vanilla gte-Qwen2 + legacy Mini-LM. Training regimes: single-stage InfoNCE vs. two-stage (InfoNCE + Triplet-NCE). Serving: quantized ANN index variants + non-linear scoring head.

### 5.2. Offline k-NN Retrieval Protocol

Precompute document embeddings → construct geo-constrained candidate sets → k-NN evaluation → aggregate by market and vertical (mean R@20, R@200).

### 5.3. Online A/B Testing Protocol

Semantic search as dedicated retrieval channel blended with lexical results → final ranking stage. Primary metrics: CVR and Search CoPSU (Orders/Clicks Per Search Unit).

### F. Non-linear Distance Function (Micro Re-ranking)

Architecture: lightweight FFN scoring head $g(\mathbf{q}, \mathbf{d}) = \text{FFN}([\mathbf{q}; \mathbf{d}])$:

$$\mathbf{h}_1 = \text{GELU}(W_1 \mathbf{z} + b_1) \quad (6)$$
$$\mathbf{h}_2 = \text{LayerNorm}(W_2 \mathbf{h}_1 + b_2) \quad (7)$$
$$s = W_3 \mathbf{h}_2 + b_3 \quad (8)$$

## 7. Conclusion and Future Work

We presented a production-oriented embedding system for Uber Eats that provides a unified, multilingual, and multi-vertical solution for search retrieval. By leveraging a two-tower Qwen2 backbone and fine-tuning with InfoNCE, we achieved substantial recall gains across 6 diverse markets. MRL enables flexible cost-quality tradeoffs. INT8 quantization at 256D MRL cut is validated as a robust, cost-efficient default for large-scale deployment.

**Future work:**
- Scale to larger backbones (7B/14B) and late-interaction layers for re-ranking
- Incorporate multi-modal signals (images) into document tower
- Move toward fully isolated-index rollout per model run
- Multi-objective learning (retrieval relevance + downstream pCVR)
